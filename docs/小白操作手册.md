# 小白操作手册（Human-like Nervous System）

适用对象：没有编程经验，只需要会看结果、会点按钮。

## 1. 项目在做什么

- 目标：让“指令发出 -> 传导 -> 反馈 -> 再判断”形成可验证闭环。
- 当前状态：已具备自动门禁、自动报告、自动告警、自动关单能力。

## 2. 你只需要看 3 个入口

1. `Actions -> Nightly Release Gate`
2. `Actions -> Precheck`
3. `docs/release-acceptance-latest.md`

## 3. 每天 3 分钟检查

1. 打开 `Nightly Release Gate`
2. 看 Summary 四行：
   - `OverallDecision`
   - `FullDrillGate`
   - `ScalabilityGate`
   - `ExternalMetricsGate`
3. 如果有异常，把这四行发给助手处理

## 4. 发布前只做一件事

看 `Precheck` 是否全绿：

- 绿色：可继续发布
- 红色：停止发布，发日志给助手

说明：`Precheck` 已是硬门禁，会自动拦截不合格发布。

## 5. 常见状态怎么理解

- `OverallDecision=WARN`：门禁未达标（不是程序崩溃）
- `ScalabilityGate=WARN`：性能/并发不达标
- `ExternalMetricsGate=WARN`：外部可用性/投递/重试不达标
- `FullDrillGate=SKIPPED`：本次没跑完整严格链路

## 6. 常用命令（会用就够）

```bash
pnpm gate:release:precheck
pnpm gate:release:strict
pnpm gate:release:strict:1000
pnpm report:release-acceptance
pnpm report:external-metrics-snapshot
```

## 8. 推荐节奏（不用写代码）

1. 第 1 周：每天看 Nightly Summary，先不改阈值
2. 第 2 周：每周看一次趋势和发布报告
3. 每次发布前：只看 `Precheck`，红灯就停

## 9. 你遇到问题时怎么发信息最有效

直接发以下任意一项即可：

1. 失败 run 的链接
2. Summary 四行
3. 报错最后 20 行日志

这样助手可以最快定位并修复。
